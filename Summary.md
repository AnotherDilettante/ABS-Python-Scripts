Here is a summary of the project state, script functions, and capabilities. You can use this as a `README.md` or handover document.

### **Project: ABS Data Automation & Visualization Pipeline**

**Goal:** Automate the extraction, cleaning, and visualization of key Australian economic indicators (GDP, CPI, Labour Force, etc.) from the Australian Bureau of Statistics (ABS).

**Tech Stack:** Python, Pandas, Matplotlib, Streamlit, Plotly, `readabs` (package).

---

### **1. Script Directory**

#### **`start.py` (The Control Center)**
*   **Function:** Terminal User Interface (TUI) that orchestrates all pipeline operations from a single interactive menu.
*   **Key Features:**
    *   Real-time status reporting for data files, charts, and dependencies
    *   One-click execution of download, chart generation, or dashboard launch
    *   Full update mode (download + charts in sequence)
    *   Built with the `rich` library for beautiful terminal output
    *   Error handling and user-friendly prompts
*   **Usage:** `python start.py`

#### **`config.py` (The Configuration)**
*   **Function:** Acts as the "Master Registry" or "Shopping List" for the project.
*   **Key Features:**
    *   Contains a dictionary list (`ABS_DATASETS`) defining the Catalogue ID, Frequency, and Tables to fetch.
    *   Supports two modes:
        1.  **Simple:** Download a table and save the default view.
        2.  **Advanced:** Download a table but filter specifically for a list of **Series IDs** (e.g., `A2304402X`).
    *   Sets global variables like `HISTORY_YEARS` (currently 10 years).

#### **`download.py` (The Extract & Load)**
*   **Function:** Iterates through `config.py` and downloads data.
*   **Key Features:**
    *   Uses `readabs` to fetch the full catalogue from the ABS website.
    *   Calculates rows to keep based on frequency (Monthly=120 rows, Quarterly=40 rows).
    *   **Filtering:** If "Advanced" mode is used in `config.py`, it strips out all columns *except* the requested Series IDs before saving.
    *   **CLI Arguments:** Supports `--freq`, `--dataset`, `--years`, `--cat` flags for selective downloads
    *   **Output:** Saves clean CSV files to `abs_data_output/`.
*   **Usage:** `python download.py` or `python download.py --freq monthly`

#### **`generate_charts.py` (The Visualizer)**
*   **Function:** Loops through the CSV files generated by `download.py` and generates standardized line charts.
*   **Key Features:**
    *   **Styling:** Uses a custom style (Red/Black lines, clean spines, "Roboto" font with "Arial" fallback).
    *   **Dynamic:** Plots specific columns if defined in `config.py`, otherwise defaults to the headline column.
    *   **Output:** Saves PNG images to `abs_charts_output/`.
*   **Usage:** `python generate_charts.py`

#### **`dashboard.py` (The Interactive Explorer)**
*   **Function:** Streamlit web dashboard for interactive data exploration.
*   **Key Features:**
    *   **Dataset selection** - Choose from any configured ABS dataset
    *   **Series toggles** - Show/hide individual time series with checkboxes
    *   **Date range slider** - Dynamically adjust the time window
    *   **Interactive charts** - Hover for precise values, zoom, pan
    *   **Latest values table** - Current data points with period-on-period changes
    *   **Output:** Runs at `http://localhost:8501`
*   **Usage:** `streamlit run dashboard.py`

#### **`s03_construction_cycle.py` (Specific Analysis)**
*   **Function:** A specialized script for cross-dataset analysis (Building Cycle).
*   **Key Features:**
    *   Demonstrates how to load **Monthly** data (Approvals) and **Quarterly** data (Commencements).
    *   Uses Pandas `.resample()` to sum Monthly data into Quarterly figures for valid comparisons.
    *   Plots multiple datasets on a single chart.
*   **Note:** This is a specialized analysis example, not part of the core pipeline.

### **2. Project Outputs**

*   **`abs_data_output/`**: Contains the raw or filtered CSV data files (ignored by Git).
*   **`abs_charts_output/`**: Contains the generated PNG charts (ignored by Git).
*   **`.readabs_cache/`**: Hidden folder where `readabs` stores raw Excel downloads to speed up subsequent runs (ignored by Git).

### **4. Quick Start**

**First Time Setup:**
```bash
# Activate virtual environment
venv\Scripts\activate  # Windows
source venv/bin/activate  # Mac/Linux

# Install dependencies (if not already done)
pip install -r requirements.txt

# Run the TUI control center
python start.py
```

**Regular Workflow:**
```bash
# Option 1: Use the interactive TUI
python start.py

# Option 2: Direct commands
python download.py                # Download latest data
python generate_charts.py         # Generate static charts
streamlit run dashboard.py        # Launch interactive dashboard
```
